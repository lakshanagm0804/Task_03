## Task 3: SQL for Data Analysis

### Objective

To use SQL queries for extracting, filtering, and analyzing data from a relational database. The aim of this task was to gain practical experience in manipulating structured data and generating meaningful insights using SQL.

---

### Tools Used

* MySQL
* MySQL Workbench

---

### Dataset Used

**Ecommerce_SQL_Database**

The database consists of multiple related tables such as:

* Customers
* Categories
* Products
* Orders
* OrderItems

These tables are connected using Primary Keys and Foreign Keys to maintain data integrity.

---

### Work Completed

During this task, the following SQL concepts were implemented:

#### 1. Data Retrieval

* Used `SELECT` statements to retrieve data.
* Applied `WHERE` conditions to filter records.
* Sorted output using `ORDER BY`.

#### 2. Grouping and Aggregation

* Used `GROUP BY` for categorizing records.
* Performed calculations using aggregate functions:

  * `SUM()`
  * `AVG()`
  * `COUNT()`
  * `MAX()`
  * `MIN()`

#### 3. Joins

Implemented joins to combine data from multiple tables:

* INNER JOIN
* LEFT JOIN
* RIGHT JOIN

#### 4. Subqueries

Created nested queries to solve advanced data analysis problems, such as identifying customers without orders and products that were never purchased.

#### 5. Views

Created SQL views to simplify repeated analysis and improve query readability.

#### 6. Query Optimization

Applied indexes to improve query execution speed and database performance.
